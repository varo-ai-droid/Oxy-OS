# Phase 2: Tournament - Round 1 Judge Scores

## Scoring Criteria (1-10 each)
1. **Pain** - How acute/urgent is the problem?
2. **Urgency** - How immediately do they need a solution?
3. **Reachability** - How easy to find/access these customers?
4. **WTP** - Are they already paying for bad solutions?
5. **Buildability** - Can a small team build MVP in 4-8 weeks?
6. **Incumbent Weakness** - Are current solutions terrible?

## Judge Notes on Winner

**Judge 1 (Pragmatist)**: "Only candidate with 10/10 pain + 10/10 urgency + 10/10 WTP + large market. Buildable with Shopify APIs + email/shipping integrations. Incumbent (Chargeflow, Midigator) use templates - evidence automation is the gap."

**Judge 2 (Revenue)**: "$19-500/mo WTP range means can start low, expand. 2M+ Shopify stores, ~30% have chargeback issues = 600k addressable. Clear path to $100k MRR."

**Judge 3 (Technologist)**: "Shopify Admin API + GraphQL + Gmail API + Shipping carrier APIs = evidence graph. Reason code mapping is deterministic. Technical moat = integrations + reason code logic."

**Judge 4 (Operator)**: "Daily pain. Merchant wakes up to chargeback notification, has 7 days. Current tools: 'upload PDF template'. This: 'we pulled tracking, emails, order history - here's the packet, approve?' No-brainer."

**Judge 5 (Skeptic)**: "Risk: Shopify could build this. But they haven't in 10 years. Risk: Low win rates persist even with evidence. But data shows evidence quality = win rate. Verdict: viable with fixes."
